## About

This is not a physics analysis tool — it is a **data platform** designed to make scientific datasets explorable by anyone, regardless of their physics background.

---

## What is ParticleSight?

ParticleSight is a full-stack web platform that takes real particle collision data from the CERN Open Data Portal and automatically discovers hidden patterns, correlations, and anomalies — then explains every finding in plain English using AI.

A researcher, student, or curious person can select a CERN dataset, click Analyse, and see:

- Statistical profiles of every data column
- The strongest correlations between physics variables
- Anomalous collision events flagged by machine learning
- Plain-English explanations of every finding generated by AI (Llama 3.1 via Cerebras)

**The gap it fills:** CERN has released over 3,000 datasets publicly. Exploring them currently requires deep physics expertise. ParticleSight removes that barrier entirely.

## How It Works

```
User selects a CERN dataset
        ↓
Backend downloads CSV from CERN Open Data Portal
(checks local cache first, downloads if not found)
        ↓
pandas loads it into a DataFrame
        ↓
┌─────────────────────────────────────┐
│         ANALYSIS PIPELINE           │
│                                     │
│  1. profiler.py                     │
│     → Stats for every column        │
│     → Histogram data (50 bins)      │
│     → Flags unusual skewness > 2.0  │
│                                     │
│  2. correlation.py                  │
│     → Spearman r for all pairs      │
│     → p-value significance test     │
│     → Returns top 10 strongest      │
│                                     │
│  3. anomaly.py                      │
│     → StandardScaler normalisation  │
│     → Isolation Forest (3% contam.) │
│     → Scores every event            │
│     → Finds most anomalous features │
└─────────────────────────────────────┘
        ↓
Results sent to Cerebras API (Llama 3.1 8B)
        ↓
AI returns 5 plain-English insights as JSON
        ↓
Everything saved to PostgreSQL (AnalysisResult table)
        ↓
React frontend renders:  [in progress]
  - Distribution histograms
  - Correlation heatmap
  - Anomaly scatter plot
  - AI insight cards
```

| Purpose | URL |
|---------|-----|

All analysis runs locally on the backend. No data is sent back to CERN.

## ML Approach

### Anomaly Detection — Isolation Forest (unsupervised)

No labels needed. The model learns what "normal" collision events look like, then flags anything that deviates significantly.

- All numeric columns used as features
- Normalised with StandardScaler before fitting
- Contamination: 3% (flags the 3% most unusual events)
- Returns anomaly scores for all events + top 5 most anomalous features

### Distribution Profiling

For every numeric column: mean, median, std, skewness, kurtosis, histogram (50 bins). Columns with skewness > 2.0 are flagged as unusual and highlighted to the AI.

### Correlation Discovery

Spearman correlation across all numeric column pairs with p-value significance testing. Returns the top 10 strongest statistically significant correlations ranked by absolute r value.

## Running Locally

### Prerequisites

- Python 3.13+
- Docker Desktop
- Node.js 20+ (for frontend, when ready)

### Backend

```bash
# 1. Start PostgreSQL
docker compose up -d

# 2. Create and activate virtual environment
cd backend
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux

# 3. Install dependencies
pip install -r requirements.txt

# 4. Create .env file (see Environment Variables below)

# 5. Start the API
uvicorn app.main:app --reload
```

Visit [http://localhost:8000/docs](http://localhost:8000/docs) for interactive API docs.

## Environment Variables

Create `backend/.env`:

